# Платы устройства

Описание трёх проектов Altium из `hardware/`: что за плата, из чего собрана и
как связана с остальными.

Состав восстановлен по схемам и библиотекам проектов. Где сведений в файлах не
нашлось, это отмечено прямо — лучше пробел, чем догадка, выданная за факт.

## Как платы связаны

```text
Head_Mouse            MAX_V5              EEG_MASTER_REV2
(на голове мыши)  →  (аналоговый тракт) →  (МК, BLE, питание)  →  BLE → ПК
   электроды           MAX30003              STM32F103 + JDY-16
```

Сигнал идёт от электродов на головном модуле в аналоговый тракт MAX30003,
оттуда по SPI в STM32, а из него по UART в модуль JDY-16 и дальше по BLE в
[приложение оператора](https://github.com/duckduckduck1/smart-sleep-eeg-app).

## EEG_MASTER_REV2 — основная плата

Несёт микроконтроллер, радиомодуль и питание. Схема разбита на четыре листа.

| лист | что на нём |
|---|---|
| `Main.SchDoc` | сборка листов, отладочный разъём SWD |
| `STM32.SchDoc` | STM32F103, модуль **JDY-16**, транслятор уровней **BSS138**, светодиод |
| `Power.SchDoc` | зарядка **TP4056**, стабилизаторы **RT9013-33GB** (3,3 В) и **RT9013-18GB** (1,8 В) |
| `Connectors.SchDoc` | разъёмы **BM10B-SRSS-TB** (JST, 10 контактов) к плате MAX30003 |

Два стабилизатора не случайны: цифровая часть MAX30003 питается от 1,8 В, а
STM32 и радиомодуль — от 3,3 В. Отсюда же и BSS138 — согласование уровней между
доменами.

Наличие TP4056 означает питание от литиевого аккумулятора с зарядкой по USB.

Файлы плат: `EEG_Master_Rev2.PcbDoc`, `EEG_MASTER_REV2(2).PcbDoc`,
`ALL_PCB.PcbDoc`. Какая из трёх версий актуальна — из файлов не следует.

## MAX_V5 — аналоговый тракт

Плата с **MAX30003**: снимает сигнал с электродов и отдаёт его по SPI.
Схема — `Sheet2.SchDoc`, плата — `PCB2.PcbDoc`.

Параметры тракта, от которых зависит перевод отсчётов в микровольты
(`V_REF = 1.0 В`, `GAIN = 160`), описаны на стороне приложения:
[docs/data-format.md](https://github.com/duckduckduck1/smart-sleep-eeg-app/blob/main/docs/data-format.md).

## EEG_Head_Mouse V1 — головной модуль

Плата, которая ставится на голову животного и держит электроды.
Схема — `Sheet1.SchDoc`, плата — `PCB1.PcbDoc`.

Из схемы состав вытащить не удалось — текстовых обозначений в файле нет. По
библиотекам проекта видно, что используются микроразъёмы **SAMTEC** серии DWM
(`DWM-03-X-X-S-X`, `DWM-04-60-G-S-625`) — типовой выбор для съёмного соединения
с головным имплантом.

## Что лежит в проектах, а что нет

В репозитории то, без чего проект не открыть и плату не повторить: схемы,
платы, файлы проектов, BOM, задания вывода и библиотеки под конкретные
компоненты.

Не хранится то, что Altium делает сам:

| не хранится | почему |
|---|---|
| `History/` | собственный контроль версий Altium — снимки тех же файлов, которые уже версионирует git |
| `Project Outputs for *` | герберы, сверловка, PDF — генерируются из проекта |
| `Project Logs for *` | журналы работы Altium |
| `__Previews/` | миниатюры предпросмотра |
| `IntegratedLibrary/` | стандартные библиотеки резисторов, конденсаторов, пассивки, транзисторов — общие, ставятся с Altium |

Из-за этого при первом открытии проекта Altium может пожаловаться на
недостающие библиотеки для типовых компонентов. На уже размещённые элементы это
не влияет: их данные лежат в самих схемах и платах. Библиотеки нужны только
чтобы **добавлять новые** типовые компоненты — тогда подключите стандартные
библиотеки Altium.
