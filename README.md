# Устройство ЭЭГ «Умного сна» — прошивка и схемотехника

Прошивка STM32F103C8 и схемотехника плат ЭЭГ-устройства для нейробиологической
лаборатории, которая исследует сон на мышах.

Устройство снимает сигнал ЭЭГ аналоговым трактом на **MAX30003**, отдаёт его по
BLE через модуль **JDY-16** и принимает обратно команды управления
фотобиомодуляцией (светом). На другом конце — приложение оператора
[smart-sleep-eeg-app](https://github.com/duckduckduck1/smart-sleep-eeg-app).

## Железо

Устройство собрано из трёх плат: головной модуль с электродами → аналоговый
тракт на MAX30003 → основная плата с микроконтроллером, радиомодулем и
питанием. Схемотехника всех трёх — в `hardware/`, разбор —
в [docs/hardware.md](docs/hardware.md).

- **МК** — STM32F103C8T6, тактирование по умолчанию из `.ioc`.
- **АЦП / аналоговый тракт** — MAX30003, подключён по SPI1.
- **BLE** — модуль JDY-16 на USART1, скорость **9600 бод**, 8N1.
- **Свет (ФБМ)** — ШИМ на TIM2, канал 3, нога PA2.
- **Питание** — литиевый аккумулятор, зарядка TP4056, стабилизаторы RT9013
  на 3,3 В (МК и радио) и 1,8 В (цифровая часть MAX30003).

### Распиновка

| нога | назначение |
|---|---|
| PA5 | SPI1_SCK — MAX30003 |
| PA6 | SPI1_MISO — MAX30003 |
| PA7 | SPI1_MOSI — MAX30003 |
| PA4 | выход GPIO, опускается в 0 при старте и больше не меняется — CS для MAX30003 (SPI NSS программный, транзакции им не обрамляются) |
| PA9 | USART1_TX — в JDY-16 |
| PA10 | USART1_RX — из JDY-16 |
| PA2 | TIM2_CH3, ШИМ светодиода ФБМ (**в `.ioc` не описана**) |
| PA1 | TIM2_CH2 — описана в `.ioc`, кодом не используется |
| PA13 / PA14 | SWDIO / SWCLK — отладка и прошивка |

## Как собирается

Схема работы сложилась такая:

1. **STM32CubeIDE** — только генератор. В `.ioc` назначаются ноги и периферия,
   по нему создаётся код инициализации на HAL.
2. **VisualGDB в Visual Studio** — то, чем проект реально собирается и шьётся.

Файл проекта `Test_eeg/Test_eeg/Test_eeg.vcxproj` собирает исходники из
корневого `Core/` (поднимаясь на два уровня вверх) плюс лежащий рядом с ним
драйвер `max30003.c`.

> **Проект STM32CubeIDE в текущем виде не собирается.** В его путях включения
> только `../Core/Inc` и HAL, а `Core/Inc/main.h` инклюдит `"max30003.h"`,
> которого в `Core/Inc` нет. Драйвер лежит в `Test_eeg/Test_eeg/` и в сборку
> CubeIDE не входит. Собирать нужно через VisualGDB.

## Обмен с приложением

### Поток отсчётов

250 Гц, один отсчёт — три байта. Прошивка копит 12 отсчётов и отдаёт их одним
пакетом в 36 байт.

Формат отсчёта, декодирование в микровольты и всё, что с этим связано, описано
на стороне приложения — дублировать не нужно:
[docs/data-format.md](https://github.com/duckduckduck1/smart-sleep-eeg-app/blob/main/docs/data-format.md).

> **Запас по скорости почти исчерпан.** 250 Гц × 3 байта = 750 Б/с, а UART на
> 9600 бод даёт 960 Б/с — занято около 78 %. Добавить в поток что-то ещё без
> повышения скорости не выйдет.

### Команда управления светом

Приходит по тому же UART, четыре байта:

```text
[on_off, pwm, 0x0D, 0x0A]
```

| байт | значение |
|---|---|
| `on_off` | `0x01` — включить ШИМ, `0x00` — выключить |
| `pwm` | скважность, **0…249** |
| `0x0D 0x0A` | обязательный конец пакета |

Разбирает команду `HAL_UART_RxCpltCallback` в `Core/Src/main.c`: байты копятся
до пары `0x0D 0x0A`, после чего пакет длиной ровно 4 байта принимается к
исполнению, остальные отбрасываются.

> **Расхождение с приложением.** Период TIM2 равен 249, то есть полная яркость —
> это `pwm = 249`, и всё, что больше, прошивка обрезает до 249. Приложение же
> считает `pwm = round(уровень × 255 / 100)`. В результате уровни 98 % и 99 %
> дают ту же яркость, что и ~97,6 %. Расхождение небольшое, но реальное:
> договориться нужно на одну сторону — либо приложение считает от 249, либо
> период таймера меняется на 255.

## Известные особенности

- **Не перегенерировать код из `.ioc`.** Часть логики дописана руками в файлах,
  которыми владеет CubeMX, и лежит вне секций `USER CODE`: в `main.c` — вся
  логика целиком (маркеры из файла удалены), в `tim.c` — настройка TIM2 канала 3
  и ноги PA2. Генерация их перезапишет. Ноги давно назначены и переназначать их
  не нужно, так что на практике это не мешает — но если понадобится, сначала
  сохраните `main.c` и `tim.c`.
- **`.ioc` описывает не то, что работает.** В нём канал 2 TIM2 на ноге PA1,
  а прошивка использует канал 3 на PA2. Расхождение историческое; источник
  правды — код, не `.ioc`.
- **Проект CubeIDE не собирается** — драйвер MAX30003 вне его сборки. Собирать
  через VisualGDB.
- **Шкала ШИМ не совпадает с приложением** — 249 против 255, см. выше.
- **Драйвер лежит в папке, похожей на отдельный проект** — из-за чего кажется,
  что проектов два. Проект один.
- **CS для MAX30003 держится прижатым постоянно.** PA4 опускается при старте и
  больше не трогается, транзакции им не обрамляются. Работает, потому что на
  шине одно устройство.
